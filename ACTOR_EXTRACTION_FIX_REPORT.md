# تقرير إصلاح خلل استخراج الفاعل

## المشكلة
كان النظام يستخرج فاعلين غير دقيقين من الأخبار، مثل:
- جمل طويلة (>15 كلمة) تُعتبر خطأ كفاعلين
- مصادر إعلامية (قناة العربية، الميادين...) تُستخرج كفاعلين
- عدم توحيد الأسماء (ترامب vs ترمب، خامنئي vs الزعيم الأعلى)
- رفض أخبار صحيحة بسبب عدم اكتمال 5W1H بشكل صارم جداً

## الحل المطبق

### 1. فئة ActorExtractorV2 الجديدة
```python
class ActorExtractorV2:
    """
    مستخرج فاعل محسّن يعالج مشاكل النسخة السابقة
    """
```

**الميزات:**
- **فهرس عكسي للقادة (LeaderIndex)**: بحث O(1) بدلاً من O(n×m)
- **قائمة سوداء للمصادر الإعلامية**: استبعاد تلقائي لـ 20+ مصدر إعلامي
- **أنماط سلبية (Passive Patterns)**: استبعاد الجمل التي تبدأ بـ "تم رصد"، "تقارير"، إلخ
- **تنظيف وتوحيد الأسماء**: تحويل "ترمب" → "دونالد ترامب"، "خامنئي" → "علي خامنئي"
- **حد أقصى لطول الفاعل**: 8 كلمات كحد أقصى

### 2. قاعدة بيانات قادة محسّنة

### 3. تحسين extract_5w1h
**توسيع الكلمات المفتاحية:**
- **What**: 40+ كلمة (هجوم، اجتماع، إعلان... مقتل، إصابة، دمار...)
- **Where**: 60+ موقع (دول، مدن، مناطق ساخنة)
- **When**: أيام الأسبوع + كلمات زمنية إضافية
- **Why**: 15+ كلمة سبب + استنتاج ضمني
- **How**: 25+ أسلوب عسكري/أمني/إعلامي

**قيم افتراضية ذكية:**
- إذا لم يُوجد فاعل واضح → "فاعل غير محسوم" (بدلاً من الرفض)
- إذا لم يُوجد وقت → "وقت النشر"
- إذا لم يُوجد موقع → "غير محدد"
- إذا لم يُوجد سبب → "سياق عام"
- إذا لم يُوجد أسلوب → "أسلوب غير محدد"

## التوصيات الإضافية

### قصيرة المدى:
1. **تحديث قاعدة البيانات**: إضافة قادة جدد حسب التطورات
2. **مراجعة يدوية**: فحص 50-100 خبر عشوائي للتأكد من الدقة
3. **ضبط الأنماط**: تحسين الأنماط بناءً على الأخطاء المتبقية

### متوسطة المدى:
1. **التعلم الآلي**: تدريب نموذج NLP للتعرف على الكيانات (NER)
2. **قاعدة معرفية**: ربط القادة بالكيانات تلقائياً
3. **API خارجي**: استخدام Google Cloud NLP أو Azure Text Analytics للتحقق

### طويلة المدى:
1. **نظام تقييم ثقة**: درجة ثقة لكل فاعل مستخرج (0-100%)
2. **تصحيح ذاتي**: تعلم من التصحيحات اليدوية
3. **دعم لغات متعددة**: الإنجليزية والفارسية والعبرية

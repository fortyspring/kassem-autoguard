# نظام التحقق من الأخبار 5W1H

## نظرة عامة

تم تطوير نظام تحقق متقدم لضمان جودة الأخبار قبل النشر، بناءً على مبدأ **5W1H** الصحفي الكلاسيكي:

- **من (Who)**: الفاعل في الحدث
- **ماذا (What)**: طبيعة الحدث
- **لماذا (Why)**: السياق أو الدافع
- **كيف (How)**: الكيفية أو الأداة المستخدمة
- **متى (When)**: وقت الحدث
- **أين (Where)**: مكان الحدث

## المشاكل المكتشفة

بناءً على تحليل البيانات المرفقة، تم رصد المشاكل التالية:

### 1. مشكلة تصنيف الفاعل
- وسائل إعلام تُصنَّف كفاعلين (مثال: "إيران الآن"، "العربية"، "الميادين")
- مصادر إخبارية تُذكر كفاعلين رئيسيين
- فاعلون غير محددين بشكل دقيق ("فاعل غير محسوم" بدون محاولة استنتاج)

### 2. مشكلة نشر الأخبار غير المستوفية للشروط
أخبار لا تجيب على الأسئلة الأساسية يتم نشرها:

| المشكلة | أمثلة من البيانات |
|---------|-------------------|
| **بدون فاعل واضح** | "18/04/2026" - فقط تاريخ بدون حدث |
| **وسائل إعلام كفاعل** | "إيران الآن | رئيس مجلس الشورى..." |
| **بدون مكان محدد** | "استهلاك النفط العالمي لخمسة أيام" |
| **بدون وقت محدد** | أخبار بدون طابع زمني |
| **محتوى ضعيف** | "أكثر من شهر من إجمالي استهلاك النفط في أوروبا" |

## الحل المقترح

### الملف الجديد: `class-5w1h-validator.php`

#### الميزات الرئيسية:

1. **التحقق من العناصر الستة**
   - كل عنصر له وزن محدد في حساب الدرجة النهائية
   - عناصر إلزامية: من، ماذا، متى، أين
   - عناصر اختيارية: لماذا، كيف

2. **كشف الفاعل الإعلامي**
   - قائمة سوداء بـ 50+ وسيلة إعلامية
   - خوارزميات لاستخراج الفاعل الحقيقي من التقارير الإعلامية
   - تصحيح تلقائي لأسماء الفاعلين

3. **نظام النقاط**
   - درجة دنيا للنشر: 70%
   - نقاط إضافية لتعدد المطابقات
   - خصم للعناصر الناقصة

4. **تقرير تفصيلي**
   - عرض حالة كل عنصر
   - اقتراحات للتحسين
   - سجل بالأسباب عند الرفض

### أمثلة على الأنماط المكتشفة

#### أنماط الفاعلين الصالحين:
```php
'حزب الله' => 'المقاومة الإسلامية (حزب الله)'
'ترامب' => 'الرئيس الأمريكي دونالد ترامب'
'لافروف' => 'وزير الخارجية الروسي'
```

#### أنماط الوسائل الإعلامية (مرفوضة كفاعلين):
```php
'إيران الآن', 'العربية', 'الميادين', 'الجزيرة', 
'رويترز', 'beiruttime', 'قناة العربية'
```

#### أنماط كشف الوقت:
```php
/\d{1,2}[\/\-]\d{1,2}[\/\-]\d{2,4}/  // 18/04/2026
/\d{1,2}:\d{2}\s*[صم]?/ui            // 01:50 م
/(اليوم|الأمس|غداً|الآن|قبل قليل)/ui
```

#### أنماط كشف المكان:
```php
/(في |إلى |من |على |قرب|جنوب|شمال|شرق|غرب)/ui
/(لبنان|سوريا|فلسطين|إيران|غزة|القدس)/ui
```

## التكامل مع النظام

### دالة الاعتراض:
```php
function sod_intercept_news_publish(array $news_data): array {
    $validation = Sod_5W1H_Validator::validate_before_publish($news_data);
    
    if (!$validation['approved']) {
        // رفض النشر وتحويل للمراجعة
        return ['success' => false, 'action' => 'blocked'];
    }
    
    // السماح بالنشر مع التصحيحات
    return ['success' => true, 'action' => 'published'];
}
```

## الإحصائيات المتوقعة

بعد تطبيق النظام:

| المقياس | قبل | بعد (متوقع) |
|---------|-----|-------------|
| نسبة الأخبار المقبولة | ~100% | ~60-70% |
| نسبة الفواعل الصحيحة | ~45% | ~90%+ |
| متوسط درجة الجودة | غير مقاس | 75%+ |
| الأخبار المرفوضة يومياً | 0 | ~30-40% |

## التوصيات

1. **تطبيق تدريجي**: البدء بتسجيل التحذيرات فقط قبل المنع الفعلي
2. **مراجعة يدوية**: إنشاء لوحة لمراجعة الأخبار المرفوضة
3. **تغذية راجعة**: تحسين القواميس بناءً على المراجعات
4. **تدريب النموذج**: استخدام البيانات المرفوضة لتحسين خوارزميات الاستنتاج
